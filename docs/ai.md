# AIモジュール (`ai.rs`)

このモジュールは、将棋AIの思考プロセスのコアロジックを実装しています。高度な探索アルゴリズムを使用して、特定の局面で最善手を見つけます。

## 主な機能

*   **アルファベータ探索:** 探索木の枝の中で、すでに見つかった手よりも悪化が保証されているものを刈り込む、非常に効率的な探索アルゴリズムです。これにより、探索プロセスが大幅に高速化されます。
*   **静止探索:** 通常の探索の最後に起動される特殊なタイプの探索です。特に駒の捕獲やその他の「強制手」を分析し、評価が一時的で不安定な状況に基づかないようにします。
*   **トランスポジションテーブル:** 以前に分析した局面の結果を保存するキャッシュです。AIが再び同じ局面に遭遇した場合、テーブルから評価を即座に取得できるため、計算時間を大幅に節約できます。
*   **キラー手:** 探索木の他の部分で強力であることが判明した手を優先するヒューリスティックです。これにより、AIは有望な手に早期に集中できます。
*   **千日手（反復）検出:** 同じ局面が複数回出現したかどうかを検出します。これは引き分けにつながる可能性があります。

## 主要な構造体

*   `ShogiAI`: AIプロセス全体を管理する主要な構造体です。指し手順序付けロジック、評価器、千日手検出器、およびトランスポジションテーブルを保持します。

## 主要なメソッド

*   `find_best_move`: AIの主要なエントリポイントです。局面、最大探索深度、およびオプションの時間制限を受け取り、見つけられる最善手を返します。
*   **`alpha_beta_search`**: アルファベータ探索アルゴリズムを実装する再帰関数です。
*   **`quiescence_search`**: 静止探索を実装する関数です。
